# PRD: User Onboarding Redesign

## Problem

New users drop off during onboarding at a 60% rate. The current 8-step
wizard is too long and asks for information that could be inferred.

## Solution

Replace the wizard with a 3-step progressive onboarding flow that collects
only essential information upfront and defers optional setup.

## User stories

- As a new user, I want to start using the product within 2 minutes so
  that I can evaluate it quickly
- As a returning user, I want to skip steps I've already completed so
  that I don't repeat setup

## Success metrics

- Onboarding completion rate: 60% -> 80%
- Time to first value: 5 min -> 2 min
- 7-day retention: 25% -> 35%

## Scope

In: signup flow, profile setup, first-run experience
Out: billing, team management, integrations setup
